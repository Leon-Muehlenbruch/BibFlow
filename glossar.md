# Glossary

| Term                    | Meaning                                                                 |
|-------------------------|-------------------------------------------------------------------------|
| **citekey**             | A short string that identifies a paper everywhere (e.g. `chuo2020insights`). Generated by Better BibTeX. |
| **BibTeX / BibLaTeX**   | Plain-text formats for bibliographies. BibLaTeX is the modern variant.  |
| **DOI**                 | Digital Object Identifier — a unique, permanent paper ID.               |
| **Zotero Connector**    | The browser extension that one-click-saves a paper into Zotero.         |
| **Better BibTeX**       | A Zotero plugin that gives you stable citekeys + auto-export of `.bib`. |
| **Zotero Integration**  | An Obsidian plugin that imports Zotero items via a Markdown template.   |
| **Templater**           | An Obsidian plugin for runnable templates (action commands).            |
| **Dataview**            | An Obsidian plugin that runs queries over your notes (like SQL for the vault). |
| **Obsidian Git**        | An Obsidian plugin that auto-commits & pushes the vault to a Git remote.|
| **OpenAlex**            | A free, no-auth bibliographic database; powers `Paper Search.md` and `Paper Metrics.md`. |
| **FWCI**                | Field-Weighted Citation Impact — a normalised citation count from OpenAlex. |
| **persist block**       | Obsidian-template syntax (`{% persist "x" %}…{% endpersist %}`) that survives re-imports. |
| **Just the Docs**       | The Jekyll theme this site uses.                                        |
