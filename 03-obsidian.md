# Obsidian setup

## Install

Download from [obsidian.md](https://obsidian.md). Open the BibFlow vault: File → Open vault → Open folder as vault → pick the `BibFlow/` folder you cloned from GitHub.

## Required community plugins

Settings → Community plugins → Browse:

| Plugin                | Why                                                        |
|-----------------------|-----------------------------------------------------------|
| **Zotero Integration** | imports papers from Zotero into the Literature Note template |
| **Templater**         | runs the action templates (search, fetch references, …)   |
| **Dataview**          | powers the queries in `0/` control notes                  |
| **Obsidian Git**      | auto-commit & push the vault to GitHub                    |
| **Juggl**             | optional — interactive local graph view                   |

After enabling, the per-plugin settings live in `.obsidian/plugins/<plugin>/data.json`. The repo ships sensible defaults; tweak as needed.

## The vault layout

```
0/
  Aufgabenstellung.md       (your supervisor brief — fill in)
  Glossary.md               (Dataview-driven glossary)
  Literature Index.md       (Dataview list of all papers)
  Paper Metrics.md          (auto-generated by fetch_metrics.py)
  Paper Search.md           (OpenAlex-backed keyword searches)
  Shared References.md      (papers cited by ≥2 of your notes)
  fetch_metrics.py          (Python: paper quality metrics)
  fetch_references.py       (Python: extract a paper`s reference list)
  fetch_search.py           (Python: keyword search via OpenAlex)

1 Literature/                (one .md per paper, citekey-named)
2 Wiki/
  Concept Notes/             (atomic ideas linked from many papers)
  Method Notes/              (procedures, protocols)
3 Writing/                   (drafts, exported chapters)
9 Orga/
  Templates/                 (the literature-note + concept-note templates)
  Templater/                 (action templates triggered by hotkey)
```

The numbered prefixes mean the folders stay sorted in the file pane: control notes first, literature second, etc.

## The Literature Note template

`9 Orga/Templates/Literature Note.md` is rendered by Zotero Integration on import. The output looks like:

```
---
citekey: chuo2020insights
title: "Insights into the Current Trends..."
authors:
  - "See Chuan Chuo"
year: "2020"
journal: "Materials"
doi: "10.3390/ma13214993"
intext: "Chuo et al., 2020"
zotero: "zotero://select/items/@chuo2020insights"
tags: [literature]
---
# Insights into the Current Trends...
**Authors:** See Chuan Chuo, …
**Year:** 2020 · *Materials* · DOI

## Abstract
…

## Highlights
### General
- <mark class="hltr-yellow">"…general highlight…"</mark> (p. 4)
### Key terms
- <mark class="hltr-green">"…key term…"</mark> (p. 7)

## My summary
{% persist "summary" %}…{% endpersist %}

## Key findings
{% persist "findings" %}…{% endpersist %}

## Connections
{% persist "connections" %}…{% endpersist %}

## Open questions
{% persist "questions" %}…{% endpersist %}

## Cited works
{% persist "references" %}…{% endpersist %}
```

The `{% persist %}` blocks survive Zotero re-imports — your own notes never get overwritten. The Highlights section is rebuilt fresh from Zotero on every re-import.

The CSS snippet at `.obsidian/snippets/zotero-highlights.css` colours the `<mark>` tags to match Zotero. Enable it in Settings → Appearance → CSS snippets.

## Templater action commands

Bound to a single hotkey via the **Launcher** template (default: Option+T). Picker shows:

```
─── Literature ───
Import Paper from Zotero
Fetch References for the current paper

Refresh All Literature
Refresh Paper Metrics
─── Search ───
Paper Search
```

| Command            | What it does                                             |
|--------------------|----------------------------------------------------------|
| Import Paper       | triggers the Zotero Integration import (same as Cmd+Shift+I) |
| Fetch References   | reads the current papers DOI, queries OpenAlex, writes a list of cited works into the note |
| Refresh All        | re-imports every paper in `1 Literature/` from Zotero    |
| Refresh Metrics    | re-runs `fetch_metrics.py` to update `Paper Metrics.md`  |
| Paper Search       | runs `fetch_search.py` against your `Paper Search.md`    |
