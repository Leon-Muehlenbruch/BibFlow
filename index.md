# BibFlow

A reproducible literature workflow for thesis and academic writing. Five tools, one pipeline:

| Tool             | Role                                                |
|------------------|-----------------------------------------------------|
| **Finder**       | files on disk                                       |
| **Word / LaTeX** | the document                                        |
| **Zotero**       | reference manager, PDF reader, annotations          |
| **Obsidian**     | atomic literature notes, knowledge graph            |
| **GitHub**       | versioned backup                                    |

You read in Zotero, take notes in Obsidian, write in Word. BibFlow is the glue: highlights flow from Zotero into Obsidian, citekeys stay in sync across tools, the vault auto-commits to GitHub.

## What you get

- A pre-configured Obsidian vault: templates, scripts, plugin defaults.
- A literature note format that survives Zotero re-imports.
- Python helpers for keyword search, paper metrics, reference extraction, and PDF tables of contents.
- Reading-status tracking, in-text citation linking, and a draft scaffold for writing.
- A Templater launcher: one hotkey for every action.
- Auto-backup with a mass-deletion guard.

## Who this is for

Single-author academic writing — bachelor, master, or PhD theses; dissertations; journal articles. The kind of work where you read 20+ papers and want to stop retyping citations, stop losing notes, and find ideas you read months ago.

Not for: 5-source term papers, real-time collaborative editing.
